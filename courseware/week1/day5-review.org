#+TITLE: Feedback: Common mistakes
#+SETUPFILE: "../include/settings.org"

* Review the indentation rules (HTML)

- =html= element in the first column
- New block-level start tag?
  - Indent by two spaces
- New block-level end tag?
  - Dedent by two spaces

** Good indentation (HTML)

#+NAME: correct-html
#+BEGIN_SRC html
<!DOCTYPE html>
<html>
  <head>
    <title>This is the title</title>
  </head>

  <body>
    <h1>A heading.</h1>

    <p>A paragraph.</p>

    <ul>
      <li>List item 1</li>
      <li>List item 2</li>
    </ul>
  </body>
</html>
#+END_SRC

** Bad indentation (HTML)

#+BEGIN_SRC html
<!DOCTYPE html>
<html>
<head>
  <title>This is the title</title>
</head>

<body>
  <h1>A heading.</h1>

  <p>A paragraph.</p>

  <ul>
    <li>List item 1</li>
    <li>List item 2</li>
  </ul>
</body>
</html>
#+END_SRC

#+BEGIN_NOTES
Problems:

- =head= and =body= are not indented.
#+END_NOTES

** Bad indentation (HTML)

#+BEGIN_SRC html
<!DOCTYPE html>
<html>
  <head>
    <title>This is the title</title>
  </head>

  <body>
    <h1>A heading.</h1>

      <p>A paragraph.</p>

      <ul>
        <li>List item 1</li>
      <li>List item 2</li>
    </ul>
  </body>
</html>
#+END_SRC

#+BEGIN_NOTES
Problems:

- =p= element is indented under the =h1=
- So is the =ul=
- Indentation of =li= elements is inconsistent
#+END_NOTES

* Review the blank line rules (HTML)

- No blank line between different indentation levels
- Exactly 1 blank line between block level elements of the same indentation
- =li= is a matter of taste

** Good blank space (HTML)

* Presentational markup

#+REVEAL_HTML: <div class="leftcol">
#+BEGIN_CENTER
Presentational =b=
#+END_CENTER
#+BEGIN_SRC html
<li><b>Deutsch</b> ...</li>
#+END_SRC
#+REVEAL_HTML: </div>

#+REVEAL_HTML: <div class="rightcol">
#+BEGIN_CENTER
Semantic =span= with =class=
#+END_CENTER
#+BEGIN_SRC html
<li><span class="lang">Deutsch</span ...</li>
#+END_SRC
#+REVEAL_HTML: </div>

#+BEGIN_NOTES
- Use semantic markup instead of presentational markup
- Style with CSS
#+END_NOTES

** Don't use presentational class names

#+REVEAL_HTML: <div class="leftcol">
#+BEGIN_SRC html
<div class="left">...</div>
#+END_SRC
#+REVEAL_HTML: </div>

#+REVEAL_HTML: <div class="rightcol">
#+BEGIN_SRC html
<div class="skills">...</div>
#+END_SRC
#+REVEAL_HTML: </div>

- You might change the presentation later

* CSS layout

- 1 blank line between rules
- Property lines indented by two spaces
- Space before the ={= and after =,=

** Good CSS layout

#+BEGIN_SRC css
h1, h2 {
  color: rgb(32, 32, 32);
  border-bottom: 1px solid rgb(32, 32, 32);
}

h1 {
  margin-top: 1.5em;
}

h2 {
  margin-top: 1em;
  text-transform: uppercase;
}
#+END_SRC

** Bad: No blank lines (CSS)

#+BEGIN_SRC css
h1, h2 {
  color: rgb(32, 32, 32);
  border-bottom: 1px solid rgb(32, 32, 32);
}
h1 {
  margin-top: 1.5em;
}
h2 {
  margin-top: 1em;
  text-transform: uppercase;
}
#+END_SRC

 border-bottom: 1px solid rgb(32, 32, 32);
